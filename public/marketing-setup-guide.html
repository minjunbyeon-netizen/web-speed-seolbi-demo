먼저 모으기

시작 전 준비물 체크리스트

이게 없으면 1단계부터 막힙니다. 작업 첫날 이것부터 모으십시오.

  • 회사 공용 구글 계정 1개 필수담당자
    GA4·GTM·서치콘솔·구글광고를 전부 이 계정으로 묶습니다. 개인 지메일로 하면 퇴사 시 날아갑니다.
  • 홈페이지 소스 수정 권한 또는 개발자 연락처 제일 중요개발자
    이 사이트는 GitHub Pages라 코드를 직접 심어야 합니다. repo 접근 권한이 있는 사람이 필요합니다.
  • 마이크로소프트 계정 (클래리티용) 담당자
  • 사업자등록증 + 사업자 카드 사장님 결재
    3단계 광고(구글·메타·네이버) 계정 인증·결제에만 필요. 1·2단계엔 불필요합니다.
  • 측정 대상이 사이트에 실제 있는지 확인 전제
    전화 버튼(tel:), 카톡 버튼(pf.kakao.com), 견적 신청서가 실제로 달려 있어야 셀 수 있습니다.

실행 순서

단계별 클릭 동선

위에서 아래로 순서대로 합니다. 1·2단계만 깔아도 측정은 시작됩니다. 광고(5)는 선택입니다.

1

측정 토대 — GA4 + GTM 설치

난이도 중약 40분담당자 + 개발자
  1. analytics.google.com 접속 → 공용 구글 계정 로그인 → 속성 만들기(웹) → 측정 ID G-XXXXXXX 발급.
  2. tagmanager.google.com → 컨테이너 만들기(웹) → 코드 2조각(<head>용·<body> 바로 뒤용) 발급.
  3. 개발자에게 요청: 이 2조각을 사이트 모든 페이지에 심어달라(GitHub Pages면 public/index.html 등).
  4. GTM 안에서 태그 새로 만들기 → "Google 애널리틱스: GA4 구성" → 위 측정 ID 입력 → 트리거 All Pages → 저장 → 제출(게시).
확인 · GA4 '실시간' 보고서에 내 접속이 1명으로 잡히면 성공입니다.
2

전화·카톡 버튼 클릭 측정 ★ 핵심

난이도 중약 30분담당자
  1. GTM → 변수 → 기본 제공 변수 중 Click URL 체크(켜기).
  2. 트리거 만들기 → "클릭 - 링크만(Just Links)" → "일부 링크 클릭" → 조건 Click URL 포함 tel: (전화용). 같은 방식으로 pf.kakao.com(카톡용) 트리거 하나 더.
  3. 태그 만들기 → "GA4 이벤트" → 이벤트 이름 call_click / kakao_click → 위 트리거 연결 → 제출.
  4. GA4 → 관리 → 이벤트 → 전환으로 표시에서 call_click·kakao_click을 켭니다.
전화·카톡은 한 번에 같이 세팅합니다(보고서 5·6번이 같은 작업). 따로 두 번 하지 않습니다.
3

견적 신청서 제출 측정 ★ 핵심

난이도 상30~60분담당자 + 개발자
  1. 먼저 폼이 실제로 제출되는지 확인. 제출 후 "감사합니다" 페이지가 뜨면 → 그 페이지 도착을 GA4 전환으로.
  2. 같은 화면에서 처리되면 → 개발자에게 제출 성공 시 dataLayer.push({event:'lead_submit'}) 한 줄 추가 요청 → GTM에서 맞춤 이벤트 트리거 lead_submit → GA4 이벤트 태그 → 전환 표시.
이 사이트 주의 · 현재 견적 폼은 데모라 실제 전송이 안 됩니다. 측정 전에 폼을 실제 접수(메일·DB·카톡 전송)되게 먼저 연결해야 합니다.
4

검색어(서치콘솔) + 히트맵(클래리티)

난이도 중약 30분담당자 + 개발자
  1. 서치콘솔: search.google.com/search-console → 사이트 등록 → 소유권 인증(HTML 파일 업로드 또는 DNS, 개발자 도움) → GA4와 연결하면 검색어가 GA4에서 보입니다.
  2. 클래리티: clarity.microsoft.com → 프로젝트 생성 → 추적 코드 발급 → 개발자에게 사이트에 심어달라 요청. (GTM으로도 심을 수 있음)
완전 무료 · 클래리티는 방문 수 제한 없이 무료입니다.
5

(선택) 광고 전환 연동 + 리타게팅

난이도 상반나절+담당자 + 사장님 결재
  1. 구글광고/메타/네이버 각각 광고 계정 생성 → 사업자·카드 인증.
  2. 각 플랫폼 전환추적 픽셀을 사이트에 심기(개발자) → 전화·견적 전환을 광고와 연결.
  3. 리타게팅: 방문자 모수(쿠키 기반)를 만들어 전화 안 한 방문자에게 배너 재노출. 하루 예산 상한을 반드시 설정.
예산 안전장치 · 각 광고 계정에서 '하루 한도'를 먼저 걸어두십시오. 안 걸면 자동 입찰이 예산을 빠르게 소진합니다.

설치 후

잘되고 있는지 — 합격선

지표보통(합격선)밑돌면
방문 → 전화·신청 전환율방문 100명당 3~5건광고가 아니라 사이트 문구·버튼부터 손봄(보고서 8·9번)
실시간 측정 작동GA4 실시간에 내 접속·클릭이 잡힘코드 심기/게시(제출) 누락 점검
전환 표시call_click·lead_submit이 전환으로 집계GA4 '전환으로 표시' 토글 확인

개선(8·9번)을 한 뒤에는 고치기 전/후의 전환율을 다시 비교하십시오. 한 번 보고 끝내지 않고, 바꾼 게 효과 있었는지 숫자로 확인하는 것이 핵심입니다.
